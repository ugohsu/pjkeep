## 概要

pjkeep は複式簿記ベースの帳簿アプリです。  
ブラウザから操作でき、データは SQLite3 ファイル（`.db`）に保存されます。

**2つの運用形態に対応しています：**

| 形態 | アクセス方法 | 認証 | 向いている使い方 |
|------|------------|------|----------------|
| ローカル版 | `http://localhost:5000`（起動した PC のみ） | なし | 個人で1台の PC で使う |
| サーバ版 | `http://サーバのLAN-IP:5000`（同一 LAN 内のどの端末からも） | なし | 複数端末で使う（端末ごとに表示 DB を切り替え可） |

---

## ローカル版：インストール・起動

### 必要なもの

- Python 3.8 以上
- pip（Python パッケージ管理ツール）

### 初回セットアップ

```bash
# 1. アプリのフォルダに移動
cd /path/to/pjkeep

# 2. 依存パッケージをインストール（初回のみ）
pip3 install -r requirements.txt

# 3. アプリを起動
python3 app.py
```

### 起動確認

ターミナルに以下のような表示が出れば起動成功です。

```
 * Running on http://127.0.0.1:5000
```

ブラウザで `http://localhost:5000` を開いてください。

### 停止方法

ターミナルで `Ctrl + C` を押すと停止します。

### 毎回の起動

```bash
cd /path/to/pjkeep
python3 app.py
```

## サーバ版：初回セットアップ

サーバ版は Docker + Nginx + HTTP で動作します。  
LAN 内のどの端末からも `http://サーバのLAN-IP:5000` でアクセスできます。  
認証なし・外部公開なしのシンプルな構成です。

### 必要なもの

- Debian 系 Linux サーバ（同一 LAN 内に常時起動）
- Docker / Docker Compose

---

### Step 1：Docker のインストール

Docker 公式スクリプトでインストールします（推奨）。

```bash
# Docker 公式スクリプトでインストール（Docker CE + Compose プラグインが入る）
curl -fsSL https://get.docker.com | sudo sh

# compose プラグインが入っているか確認
docker compose version
# → "Docker Compose version v2.x.x" と表示されれば OK
# → 表示されない場合は以下を実行
sudo apt-get install -y docker-compose-plugin

# 現在のユーザーを docker グループに追加
sudo gpasswd -a $USER docker
# ※ usermod が見つからない場合（最小インストールで /usr/sbin が PATH に含まれない場合）は
#   gpasswd を使う。usermod が使える環境では sudo usermod -aG docker $USER でも同じ。

# 一度ログアウトして再ログインすると sudo なしで使えるようになる
# グループが反映されたか確認
groups $USER  # 出力に "docker" が含まれていれば OK
```

> **補足：apt の `docker.io` を使っている場合**  
> `docker.io` は Debian/Ubuntu の公式パッケージで、`docker-compose-plugin` には対応していません。  
> `docker-compose`（ハイフン版）を使ってください。
> ```bash
> sudo apt-get install -y docker-compose
> ```
> このマニュアルの `docker compose` コマンドはすべて `docker-compose` に読み替えてください。  
> サーバへの新規インストールであれば、curl スクリプトによる Docker CE を推奨します。

---

### Step 2：アプリファイルをサーバに配置

```bash
# git clone またはファイルを scp/rsync でコピー
git clone https://github.com/yourrepo/pjkeep.git /path/to/pjkeep
cd /path/to/pjkeep
```

---

### Step 3：.env ファイルを作成

```bash
cp .env.example .env
nano .env
```

`SECRET_KEY` をランダムな文字列に変更します：

```bash
SECRET_KEY=ここをランダムな文字列に変更する
```

**SECRET_KEY の生成：**

```bash
python3 -c "import secrets; print(secrets.token_hex(32))"
```

---

### Step 4：Docker Compose で起動

```bash
docker compose up -d
```

起動確認：

```bash
docker compose ps       # 全サービスが "Up" になっていること
docker compose logs -f  # ログをリアルタイム表示（Ctrl+C で終了）
```

---

### Step 5：アクセス確認

サーバの LAN IP を調べます：

```bash
ip addr show | grep 'inet ' | grep -v '127.0.0.1'
```

表示された IP（例：`192.168.1.100`）をブラウザで開きます：

```
http://192.168.1.100:5000
```

アプリのトップ画面が表示されれば成功です。

---

### サーバの再起動時の自動起動

```bash
# Docker デーモン自体の自動起動を有効にする
sudo systemctl enable docker

# docker-compose.yml に restart: unless-stopped が設定されているため
# Docker が起動すればアプリも自動で起動します
```

## 初期設定

アプリを初めて開くと、自動的に **設定ページ（/init）** に移動します。  
設定ページはナビバー右端の **「設定」** リンクからいつでも開けます。

### データベースを選択する場合

設定ページ上部の「**データベースを選択**」には、`data/` フォルダ内の `.db` ファイルが一覧表示されます。  
使いたいファイルの **「選択」** ボタンをクリックするだけで切り替えできます。  
現在使用中のファイルには「**使用中**」バッジが表示されます。

各ファイルの横にある **✎（鉛筆）ボタン** をクリックすると、そのDBの説明文をいつでも編集できます。

### 新規データベースを作成する場合

1. 「**新規データベースを作成**」セクションのファイル名欄に名前を入力します
   - 空欄のままにすると `pjkeep.db` として作成されます
   - **ファイル名のみ**指定します（`data/` フォルダ内に自動的に作成されます。フルパスは指定できません）
2. 「**説明文**」欄に、このDBの用途・年度などを入力できます（任意。後から変更可能）
3. 「初期勘定科目を自動登録する」チェックボックスをオンにすると、基本的な科目（現金・普通預金・未払金・預り金・基本金・繰越金・保育料収入・補助金収入・人件費・事業費）が自動で登録されます
4. **「新規作成して開始」** ボタンをクリック

### 既存の .db ファイルをアップロードして開く場合

別の環境で作成した `.db` ファイルを取り込む場合：

1. 「**既存の .db ファイルをアップロードして開く**」セクションの「ファイルを選択」で `.db` ファイルを選択します
2. 「**説明文**」欄に用途などを入力できます（任意）
3. **「アップロードして開始」** ボタンをクリック

アップロードされたファイルは `data/` フォルダに保存されます。同名ファイルが既に存在する場合はエラーになります。

> **設定はどこに保存される？**  
> どの DB を開いているかは **ブラウザの Cookie（`active_db`）** に保存されます。  
> そのため、ユーザ（ブラウザ）ごとに異なる DB を選択でき、他のユーザの表示には影響しません。  
> DB ファイルの説明文はサーバ側の `config.json` に保存されます。  
> Cookie が未設定の初回アクセス時は、`config.json` の `db_path`（従来の設定値）にフォールバックします。

## 記帳（仕訳入力）

ナビバーの **「記帳」** をクリックします。

### 基本的な入力手順

1. **取引日** を選択します（デフォルトは今日の日付）
2. **備考** に取引の内容を入力します（例：スーパー購入）
3. 仕訳行を入力します
   - **借/貸**：「借方」または「貸方」を選択
   - **勘定科目**：科目名を入力（途中まで打つとサジェストが出ます）
   - **金額**：半角数字で入力
4. **「記帳する」** ボタンをクリック

### 行の追加

**「＋ 行追加」** ボタンで行を増やせます。3行以上の複合仕訳にも対応しています。

### 金額の自動補完

1行だけ金額を空欄にしておくと、他の行から自動計算して補完されます（黄色でハイライト表示）。

**例**：食費 3,000円を現金で支払った場合
| 借/貸 | 勘定科目 | 金額 |
|-------|---------|------|
| 借方  | 食費    | 3,000 |
| 貸方  | 現金    | （空欄→自動補完） |

### 貸借バランスの確認

テーブル下部に借方合計・貸方合計がリアルタイム表示されます。
- 緑色：貸借一致（記帳可能）
- 赤色：貸借不一致（記帳不可）

### 記帳後

記帳が成功すると、同じ画面の右側（PC）または下部（スマホ）に確認テーブルが表示されます。  
「仕訳帳を確認」リンクから仕訳帳一覧に移動できます。

## 仕訳帳

ナビバーの **「仕訳帳」** をクリックします。

### 表示月の切替

画面上部のプルダウンで表示する月を切り替えます。  
記帳データがある月のみ選択肢に表示されます。

### 取引の確認

各行には取引日・備考・借方/貸方の科目名・金額が表示されます。  
**「詳細」** ボタンで各行の明細を展開表示できます。

### 取引の削除

**「削除」** ボタンをクリックすると確認ダイアログが出て、その取引の全行（複合仕訳を含む）をまとめて削除します。

> **注意**：削除したデータは元に戻せません。  
> 誤記帳の場合は、削除してから再入力してください。

### TSV エクスポート

画面右上の **「TSV エクスポート」** ボタンには3つのモードがあります。

| モード | 説明 | ファイル名 |
|--------|------|----------|
| 指定月のみ | プルダウンで選択中の月の仕訳 | `journal_YYYY-MM.tsv` |
| 全仕訳 | DB に登録されているすべての仕訳 | `journal_all.tsv` |
| 任意の期間 | 開始日〜終了日を指定してエクスポート | `journal_YYYY-MM-DD_YYYY-MM-DD.tsv` |

出力列：

| 列 | 内容 |
|----|------|
| 取引日 | YYYY-MM-DD 形式 |
| 備考 | 取引の備考 |
| 取引番号 | ファイル内での連番（複合仕訳は同じ番号） |
| 借貸 | 借方 / 貸方 |
| 勘定科目 | 科目名 |
| 金額 | 整数（円） |

Google スプレッドシートへのインポート手順：「ファイル」→「インポート」→「アップロード」→ 区切り文字「タブ」を選択。

## 財務諸表

ナビバーの **「財務諸表」** をクリックします。

### 表示月の切替

画面上部のプルダウンで月を選択します。

### 損益計算書（PL）

選択した月の収益・費用・当期純利益が表示されます。

| 項目 | 内容 |
|------|------|
| 収益合計 | 選択月の収益科目合計 |
| 費用合計 | 選択月の費用科目合計 |
| 当期純利益 | 収益合計 − 費用合計（マイナスの場合は赤字） |

### 貸借対照表（BS）

選択した月末時点の**累計残高**が表示されます。  
（その月だけでなく、記録開始からその月末までの全仕訳を集計します）

| 項目 | 内容 |
|------|------|
| 資産合計 | 資産科目の累計残高 |
| 負債合計 | 負債科目の累計残高 |
| 純資産合計 | 純資産科目の累計残高 ＋ 累計純損益 |
| 累計純損益 | 記録開始から選択月末までの収益−費用の累計 |

> **貸借差額の警告**  
> 資産合計 ≠ 負債合計 ＋ 純資産合計 の場合、表の下部に警告が表示されます。  
> 仕訳に誤りがないか確認してください。

### TSV エクスポート

画面右上の **「TSV エクスポート」** ボタンには2つのモードがあります。

| モード | 説明 | ファイル名 |
|--------|------|----------|
| 指定月のみ | 選択月の PL ＋ 月末時点の BS | `report_YYYY-MM.tsv` |
| 任意の期間 | 開始日〜終了日の PL ＋ 終了日時点の BS | `report_YYYY-MM-DD_YYYY-MM-DD.tsv` |

Google スプレッドシートへのインポート手順は仕訳帳と同様です。

## 勘定科目マスタ

ナビバーの **「勘定科目」** をクリックします。

### 科目の一覧

資産・負債・純資産・収益・費用の区分ごとに科目が表示されます。  
各科目には **科目名**、**コード**（英数字の識別子）、**表示順** があります。

### 科目の追加（1件）

**「＋ 科目を追加」** ボタンからモーダルウィンドウが開きます。

| 入力項目 | 説明 |
|---------|------|
| 科目名 | 表示に使う名前（例：食費）|
| コード | 英数字の識別コード（例：food）。自動生成されますが変更可能。|
| 区分 | 資産 / 負債 / 純資産 / 収益 / 費用 から選択 |
| 表示順 | 小さい数字が上に表示される（同一区分内での並び順）|

### 科目の一括追加（バッチ）

**「バッチ追加」** ボタンから、JSON 配列形式で複数科目をまとめて登録できます。

```json
[
  { "name": "交通費",  "code": "travel",      "element": "expenses",  "sort_order": 30 },
  { "name": "雑収入",  "code": "misc_income", "element": "revenues",  "sort_order": 30 },
  { "name": "長期借入金", "code": "loan",     "element": "liabilities", "sort_order": 30 }
]
```

**フィールド定義：**

| フィールド | 必須 | 型 | 説明 |
|-----------|------|----|------|
| `name` | 必須 | 文字列 | 科目名。DB 内で一意である必要があります |
| `code` | 必須 | 文字列 | 識別コード（英数字・アンダーバー推奨）。DB 内で一意 |
| `element` | 必須 | 文字列 | 区分（下表参照） |
| `sort_order` | 任意 | 整数 | 表示順（省略時は 0） |

**`element` の値：**

| 値 | 意味 |
|----|------|
| `assets` | 資産 |
| `liabilities` | 負債 |
| `equity` | 純資産 |
| `revenues` | 収益 |
| `expenses` | 費用 |

既に同名・同コードの科目が存在する行はスキップされ、他の行は登録されます。

### 科目の編集

各科目行の **「編集」** ボタンから変更できます。

### 科目の削除

**「削除」** ボタンをクリックします。  
その科目に仕訳が1件でも存在する場合は削除できません。

## バッチインポート（仕訳の一括取込）

ナビバーの **「バッチインポート」** をクリックします。

クレカ・銀行明細の自動仕訳、前期末の期首残高入力、給与仕訳など、まとめて取り込みたい場面で使います。  
**AI に元データ（明細 CSV・PDF など）と下記の JSON 仕様を渡すことで、自動的に仕訳 JSON を生成させることができます。**

### 操作の流れ

1. AI などで仕訳 JSON を作成する（後述）
2. バッチインポート画面の JSON テキストエリアに貼り付けるか、ファイルを選択する
3. **「プレビュー」** ボタンをクリック → バリデーション結果と仕訳内容が表示される
4. 取り込みたい仕訳にチェックを入れる（エラーのある仕訳は自動的に除外）
5. **「インポート実行」** ボタンをクリック → チェックした仕訳だけ DB に登録される

### JSON フォーマット仕様

#### 全体構造

```json
{
  "transactions": [
    {
      "date": "2026-03-15",
      "note": "Amazon 事務用品",
      "lines": [
        { "account_code": "operating", "debit_credit": "debit",  "amount": 3300 },
        { "account_code": "payable",   "debit_credit": "credit", "amount": 3300 }
      ]
    }
  ]
}
```

#### `transactions[]`（仕訳1件）のフィールド

| フィールド | 必須 | 型 | 説明 |
|-----------|------|----|------|
| `date` | 必須 | 文字列 | 仕訳日。`YYYY-MM-DD` 形式 |
| `note` | 推奨 | 文字列 | 摘要。仕訳帳の「備考」に入る |
| `lines` | 必須 | 配列 | 仕訳行（2行以上） |
| `_comment` | 任意 | 文字列 | AI の判断根拠など。DB には保存されず、プレビュー画面に表示されるだけ |
| `_confidence` | 任意 | 数値 | AI の確信度 0.0〜1.0。0.8 未満の場合はプレビューで黄色ハイライト表示。省略時は 1.0 扱い |

#### `lines[]`（仕訳行）のフィールド

| フィールド | 必須 | 型 | 説明 |
|-----------|------|----|------|
| `account_code` | 必須 | 文字列 | 勘定科目のコード（「勘定科目マスタ」画面の「コード」列の値） |
| `debit_credit` | 必須 | 文字列 | `"debit"`（借方）または `"credit"`（貸方） |
| `amount` | 必須 | 整数 | 金額（円）。1 以上の正の整数 |

> **`account_code` について**  
> 科目名（例：`事業費`）や科目 ID ではなく、コード（例：`operating`）で指定します。  
> コードは「勘定科目」画面の一覧で確認できます。

#### バリデーションルール（アプリが検証する）

1. `date` が `YYYY-MM-DD` 形式である
2. `lines` が 2 行以上ある
3. 各 `account_code` が勘定科目マスタに存在する
4. 各 `debit_credit` が `"debit"` または `"credit"` である
5. 各 `amount` が 1 以上の整数である
6. 1仕訳内の借方合計 = 貸方合計（複式簿記の大原則）

いずれかのルールに違反した仕訳はエラー表示され、チェックボックスが無効化されます（取り込みできません）。

### AI に渡すセット

| 渡すもの | 内容 |
|---------|------|
| このマニュアルの「バッチインポート」セクション | JSON フォーマット仕様とバリデーションルール |
| 勘定科目一覧 | 「勘定科目」画面に表示されているコードと名前の対応表（コピー&ペーストして渡す）|
| 元データ | クレカ明細 CSV・銀行明細・給与台帳・前期末 BS など |

`account_code` は名前ではなくコードで指定する必要があります。  
勘定科目一覧を渡すことで AI が正しいコードを使えます。

### プレビュー画面の見方

| 表示 | 意味 |
|------|------|
| 緑枠（チェック済み） | バリデーション通過。インポート対象 |
| 赤枠「エラー」バッジ | バリデーション失敗。チェック不可 |
| 黄枠「確信度 XX%」バッジ | `_confidence` が 0.8 未満。AI が自信を持てなかった仕訳。内容を確認してください |

### 使用例

#### クレカ明細を仕訳する場合

AI に渡すプロンプト例：


## 期首残高の入力方法

新しく帳簿をつけ始めるとき、または年度をまたぐときに既存の残高を登録する方法です。

「前期繰越」として仕訳を入力することで記録します。  
バッチインポートを使うと、複数科目の残高をまとめて入力できます。

### 例：普通預金 500,000円 の残高から開始する場合

| 借/貸 | 勘定科目 | 金額 |
|-------|---------|------|
| 借方  | 普通預金  | 500,000 |
| 貸方  | 基本金   | 500,000 |

- 備考欄に「前期繰越」と入力しておくと分かりやすくなります
- 取引日は期首（例：2026-04-01）にします

### 複数の資産・負債がある場合

まとめて1つの仕訳として入力できます。

| 借/貸 | 勘定科目 | 金額 |
|-------|---------|------|
| 借方  | 普通預金  | 500,000 |
| 借方  | 現金     | 30,000 |
| 貸方  | 未払金   | 20,000 |
| 貸方  | 基本金   | 510,000 |

### バッチインポートで期首残高を入力する例

```json
{
  "transactions": [
    {
      "date": "2026-04-01",
      "note": "前期繰越",
      "lines": [
        { "account_code": "bank",     "debit_credit": "debit",  "amount": 500000 },
        { "account_code": "cash",     "debit_credit": "debit",  "amount": 30000  },
        { "account_code": "payable",  "debit_credit": "credit", "amount": 20000  },
        { "account_code": "capital",  "debit_credit": "credit", "amount": 510000 }
      ]
    }
  ]
}
```

## データのエクスポート

仕訳帳・財務諸表の各ページの **「TSV エクスポート」** ボタンからダウンロードできます。  
ファイルはタブ区切り・UTF-8（BOM なし）形式です。

### 仕訳帳エクスポートのモード

| モード | 内容 | ファイル名 |
|--------|------|----------|
| 指定月のみ | プルダウン選択中の月の仕訳 | `journal_YYYY-MM.tsv` |
| 全仕訳 | DB 内のすべての仕訳 | `journal_all.tsv` |
| 任意の期間 | 開始日〜終了日を指定 | `journal_YYYY-MM-DD_YYYY-MM-DD.tsv` |

### 財務諸表エクスポートのモード

| モード | PL の範囲 | BS の基準日 | ファイル名 |
|--------|---------|-----------|----------|
| 指定月のみ | 選択月 | 選択月末 | `report_YYYY-MM.tsv` |
| 任意の期間 | 開始日〜終了日 | 終了日時点 | `report_YYYY-MM-DD_YYYY-MM-DD.tsv` |

### Google スプレッドシートへの取り込み手順

1. Google スプレッドシートを新規作成する
2. 「ファイル」→「インポート」をクリック
3. 「アップロード」タブからダウンロードした `.tsv` ファイルを選択
4. 区切り文字を **「タブ」** に設定してインポート

## データのバックアップ・移行

### ローカル版のバックアップ

`.db` ファイルをコピーするだけです。

```bash
cp /path/to/pjkeep.db /path/to/backup/pjkeep_20260401.db
```

### サーバ版のバックアップ（DB ダウンロード）

設定ページ（/init）の上部「現在のDB」欄にある **「↓ DBダウンロード」** ボタンをクリックすると、  
現在使用中の SQLite ファイルをダウンロードできます。  
ファイル名は `{DBファイル名}_{YYYYMMDD}.db` 形式になります。

ダウンロードした `.db` ファイルはローカル版 pjkeep でそのまま開くことができます。

### サーバ側のバックアップ（サーバ管理者向け）

DB ファイルはサーバの `data/` ディレクトリに保存されています。  
このディレクトリをまるごとコピーするだけでバックアップできます。

```bash
cp -r /path/to/pjkeep/data /path/to/backup/data_20260401
```

### 別の PC への移行（ローカル版）

1. 元の PC の設定ページ（/init）の「↓ DBダウンロード」ボタンで `.db` ファイルをダウンロードします
2. 新しい PC で `python3 app.py` を起動します
3. 設定ページ（/init）の「**既存の .db ファイルをアップロードして開く**」からダウンロードしたファイルを選択してアップロードします

## トラブルシューティング

### 【サーバ版】ブラウザで繋がらない

1. サーバの LAN IP を確認する：
   ```bash
   ip addr show | grep 'inet ' | grep -v '127.0.0.1'
   ```
2. Docker コンテナが起動しているか確認：
   ```bash
   docker compose ps
   docker compose logs nginx
   ```
3. ブラウザのアドレスバーに `http://IPアドレス:5000` の形式で入力しているか確認  
   （`https://` や `:5000` の省略、`http://` の省略などでアクセスできないことがあります）

### 【サーバ版】`docker compose` が使えない（`unknown command` エラー）

`docker: 'compose' is not a docker command` のようなエラーが出る場合は、Docker Compose プラグインが未インストールです。

```bash
sudo apt-get install -y docker-compose-plugin
```

インストール後に `docker compose version` を実行してバージョンが表示されれば OK です。

---

### 【サーバ版】Docker が起動しない

```bash
# ログを確認
docker compose logs app
docker compose logs nginx

# .env ファイルの内容を確認（SECRET_KEY が空でないか）
cat .env
```

### アプリが起動しない（ローカル版）

```
ModuleNotFoundError: No module named 'flask'
```
→ 依存パッケージがインストールされていません。
```bash
pip3 install -r requirements.txt
```

### ブラウザで開けない（ローカル版）

- `http://localhost:5000` または `http://127.0.0.1:5000` を試してください
- ターミナルでアプリが起動中か確認してください（`python3 app.py` が実行中であること）
- ポート 5000 が別のアプリで使用されている場合は、`app.py` 末尾の `port=5000` を別の番号に変更してください

### 「DBが設定されていません」エラー

- 設定ページ（/init）からDBを作成または指定してください
- `config.json` が削除されている場合も同じ手順で再設定できます

### 勘定科目が削除できない

仕訳に使用されている科目は削除できません。  
その科目を使っている仕訳をすべて削除してから、再度お試しください。

### バッチインポートで「account_code が存在しません」エラーが出る

AI が使用したコードが勘定科目マスタに登録されていません。  
「勘定科目」画面で該当のコードを確認し、必要であれば科目を追加してから再度インポートしてください。

### 貸借差額の警告が出る

BS ページに「貸借差額」警告が出た場合、仕訳に誤りがある可能性があります。

1. 仕訳帳を確認し、借方合計 ≠ 貸方合計 の取引がないか確認してください  
   （正常に登録された仕訳は必ず貸借が一致していますが、期首残高の入力ミスなどが原因になることがあります）
2. 期首残高が正しく入力されているか確認してください
