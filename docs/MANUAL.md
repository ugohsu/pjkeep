## 概要

pjkeep は複式簿記ベースの帳簿アプリです。  
ブラウザから操作でき、データは SQLite3 ファイル（`.db`）に保存されます。

**2つの運用形態に対応しています：**

| 形態 | アクセス方法 | 向いている使い方 |
|------|------------|----------------|
| ローカル版 | `http://localhost:5000`（起動した PC のみ） | 個人で1台の PC で使う |
| サーバ版 | `http://サーバのLAN-IP:5000`（同一 LAN 内のどの端末からも） | 複数端末・複数ユーザーで使う |

**認証について：**

どちらの形態でも、ユーザー/パスワードによるログインが必要です。  
初回アクセス時のみ、管理者アカウントの作成画面（`/setup`）が表示されます。

| ロール | できること |
|--------|----------|
| admin（管理者） | 全プロジェクトの閲覧・編集・削除、全ユーザーの一覧・削除、プロジェクトへの参加ユーザーと権限の管理 |
| user（一般ユーザー） | 自分が作成したプロジェクト、および admin から権限を付与されたプロジェクトへのアクセス（read: 閲覧のみ／write: 編集も可）、自分のアカウント削除 |

## ローカル版：インストール・起動

### 必要なもの

- Python 3.8 以上
- pip3（Python パッケージ管理ツール）

### 初回セットアップ

```bash
# 1. アプリのフォルダに移動
cd /path/to/pjkeep

# 2. 依存パッケージをインストール（初回のみ）
pip3 install -r requirements.txt

# 3. アプリを起動
python3 app.py
```

### 起動確認

ターミナルに以下のような表示が出れば起動成功です。

```
 * Running on http://127.0.0.1:5000
```

ブラウザで `http://localhost:5000` を開いてください。  
初回アクセス時は管理者アカウント作成画面（`/setup`）が表示されます。

### 停止方法

ターミナルで `Ctrl + C` を押すと停止します。

### 毎回の起動

```bash
cd /path/to/pjkeep
python3 app.py
```

## サーバ版：初回セットアップ

サーバ版は Docker + Nginx + HTTP で動作します。  
LAN 内のどの端末からも `http://サーバのLAN-IP:5000` でアクセスできます。

### 必要なもの

- Debian 系 Linux サーバ（同一 LAN 内に常時起動）
- Docker / Docker Compose

---

### Step 1：Docker のインストール

Docker 公式スクリプトでインストールします（推奨）。

```bash
# Docker 公式スクリプトでインストール（Docker CE + Compose プラグインが入る）
curl -fsSL https://get.docker.com | sudo sh

# compose プラグインが入っているか確認
docker compose version
# → "Docker Compose version v2.x.x" と表示されれば OK
# → 表示されない場合は以下を実行
sudo apt-get install -y docker-compose-plugin

# 現在のユーザーを docker グループに追加
sudo gpasswd -a $USER docker
# ※ usermod が使える環境では sudo usermod -aG docker $USER でも同じ。

# 一度ログアウトして再ログインすると sudo なしで使えるようになる
groups $USER  # 出力に "docker" が含まれていれば OK
```

> **補足：apt の `docker.io` を使っている場合**  
> `docker.io` は Debian/Ubuntu の公式パッケージで、`docker-compose-plugin` には対応していません。  
> `docker-compose`（ハイフン版）を使ってください。
> ```bash
> sudo apt-get install -y docker-compose
> ```
> このマニュアルの `docker compose` コマンドはすべて `docker-compose` に読み替えてください。  
> サーバへの新規インストールであれば、curl スクリプトによる Docker CE を推奨します。

---

### Step 2：アプリファイルをサーバに配置

```bash
git clone https://github.com/yourrepo/pjkeep.git /path/to/pjkeep
cd /path/to/pjkeep
```

---

### Step 3：.env ファイルを作成

```bash
cp .env.example .env
nano .env
```

**`SECRET_KEY`** をランダムな文字列に変更します：

```bash
# SECRET_KEY の生成
python3 -c "import secrets; print(secrets.token_hex(32))"
```

**`APP_UID` / `APP_GID`** をホストユーザーの値に設定します。  
これを設定しないと、コンテナが作成するファイル（帳簿・認証 DB）がサーバ側で root 所有になり、ローカル運用に切り替えた際に書き込みができなくなります。

```bash
# UID/GID を確認して .env に追記するワンライナー
echo "APP_UID=$(id -u)" >> .env
echo "APP_GID=$(id -g)" >> .env
```

設定後の `.env` の例：

```dotenv
SECRET_KEY=a1b2c3d4e5f6...（32バイトの乱数）
APP_UID=1001
APP_GID=1001
```

---

### Step 4：Docker Compose で起動

```bash
docker compose up -d
```

起動確認：

```bash
docker compose ps       # 全サービスが "Up" になっていること
docker compose logs -f  # ログをリアルタイム表示（Ctrl+C で終了）
```

---

### Step 5：アクセス確認と管理者アカウント作成

サーバの LAN IP を調べます：

```bash
ip addr show | grep 'inet ' | grep -v '127.0.0.1'
```

表示された IP（例：`192.168.1.100`）をブラウザで開きます：

```
http://192.168.1.100:5000
```

初回アクセス時は **管理者アカウント作成画面（`/setup`）** が表示されます。  
ユーザー名とパスワード（6文字以上）を入力して管理者アカウントを作成してください。  
以降はログイン画面（`/login`）が入口になります。

---

### サーバの再起動時の自動起動

```bash
sudo systemctl enable docker
# docker-compose.yml に restart: unless-stopped が設定されているため
# Docker が起動すればアプリも自動で起動します
```

---

### アプリを更新するとき

```bash
docker compose build app
docker compose up -d
```

> **`data/` ディレクトリのデータは消えません。**  
> `docker-compose.yml` でホスト側の `./data` をマウントしているため、再ビルド・再起動しても DB ファイルおよびユーザー情報（`users.db`）は保持されます。

## アカウント管理

### 初回：管理者アカウントの作成

`data/users.db` が存在しない（またはユーザーが 0 人の）状態でアクセスすると、自動的に `/setup` へ誘導されます。  
最初に登録したユーザーが自動的に **admin** ロールになります。

### ログイン・ログアウト

- ログイン画面（`/login`）でユーザー名とパスワードを入力します
- ナビバー右端のユーザー名をクリック → 「ログアウト」で任意のタイミングでログアウトできます

### 新規ユーザー登録

ログイン画面下部の「新規登録」リンク、または `/register` から誰でも自己登録できます。  
自己登録したユーザーは **user** ロールになります（admin への昇格は現在 DB を直接編集する必要があります）。

### プロフィール・パスワード変更

ナビバーのユーザー名 → **「プロフィール」** から変更できます。

- **パスワード変更**：現在のパスワードと新しいパスワード（6文字以上）を入力
- **アカウント削除**：自分のアカウントと、自分が所有するすべてのプロジェクトを削除します。この操作は取り消せません

### 管理者によるユーザー管理

admin ロールのユーザーは、ナビバーの **「ユーザー管理」** からユーザー一覧を確認し、任意のユーザーを削除できます。  
削除すると、そのユーザーが所有するすべてのプロジェクト（帳簿ファイル）も同時に削除されます。  
自分自身は削除できません。

ナビバーの **「プロジェクト管理」** からは、全プロジェクトの一覧確認・削除・メンバーへの権限付与が行えます。詳細は「[admin によるプロジェクト管理](#admin-によるプロジェクト管理)」を参照してください。

## 初期設定（プロジェクト管理）

ログイン後、帳簿ファイルが未選択の場合は自動的に **設定ページ（/init）** に移動します。  
設定ページはナビバー右端の **「設定」** リンクからいつでも開けます。

プロジェクト（帳簿ファイル）はユーザーごとに管理されます。  
自分が作成・アップロードしたプロジェクト、および admin から参加権限を付与されたプロジェクトが表示されます（admin は全件表示）。

### データベースを選択する場合

設定ページ上部の「**データベースを選択**」には、自分のプロジェクトが一覧表示されます。  
使いたいファイルの **「選択」** ボタンをクリックするだけで切り替えできます。  
現在使用中のファイルには「**使用中**」バッジが表示されます。

各ファイルの横にある **✎（鉛筆）ボタン** をクリックすると、そのDBの説明文をいつでも編集できます。

### 新規データベースを作成する場合

1. 「**新規データベースを作成**」セクションのファイル名欄に名前を入力します
   - 空欄のままにすると `pjkeep.db` として作成されます
   - **ファイル名のみ**指定します（`data/` フォルダ内に自動的に作成されます）
2. 「**説明文**」欄に用途・年度などを入力できます（任意。後から変更可能）
3. 「初期勘定科目を自動登録する」チェックボックスをオンにすると、基本的な科目が自動登録されます
4. **「新規作成して開始」** ボタンをクリック

作成したプロジェクトは自動的にあなたの所有になります。

### 既存の .db ファイルをアップロードして開く場合

別の環境で作成した `.db` ファイルを取り込む場合：

1. 「**既存の .db ファイルをアップロードして開く**」セクションの「ファイルを選択」で `.db` ファイルを選択します
2. 「**説明文**」欄に用途などを入力できます（任意）
3. **「アップロードして開始」** ボタンをクリック

アップロード時、システムは新規 DB を作成し、アップロードされたファイルから勘定科目・仕訳・振替データをインポートします。これにより、DB のバージョン差異やジャーナルモードの違いに依存せず安全に取り込めます。取り込み後のプロジェクトはあなたの所有として登録されます。

## admin によるプロジェクト管理

admin ロールのユーザーのみが行える操作です。ナビバーの **「プロジェクト管理」** から開きます。

### プロジェクトの削除

一覧の **「削除」** ボタンをクリックすると確認ダイアログが開きます。  
プロジェクト名を入力して確認することで、帳簿ファイル（`.db`）ごと削除されます。  
**この操作は取り消せません。削除前に DB ダウンロードでバックアップを取ることを推奨します。**

### メンバーへの権限付与・管理

一覧の **「メンバー」** ボタンをクリックするとメンバー管理モーダルが開きます。

| 操作 | 説明 |
|------|------|
| ユーザーの追加 | モーダル下部でユーザーと権限（read / write）を選択して「追加」 |
| 権限の変更 | 「read に変更」または「write に変更」ボタンをクリック |
| メンバーの削除 | 「削除」ボタンをクリック（そのユーザーはプロジェクトにアクセスできなくなります） |

**権限レベル：**

| 権限 | できること |
|------|----------|
| `read`（閲覧） | 仕訳・元帳・財務諸表の閲覧、DB ダウンロード |
| `write`（編集） | 上記に加え、仕訳・勘定科目の追加・編集・削除、バッチインポート、振替の登録・削除 |

- 権限の付与・変更・削除は admin のみが行えます
- プロジェクト作成者（owner）も `write` 権限で自動登録されます。admin は owner の権限も変更・削除できます

> **設定はどこに保存される？**  
> どの DB を開いているかは **ブラウザの Cookie（`active_db`）** に保存されます。  
> そのため、ユーザ（ブラウザ）ごとに異なる DB を選択でき、他のユーザの表示には影響しません。  
> ユーザー情報とプロジェクトのメタデータはサーバ側の `data/users.db` に保存されます。

## 記帳（仕訳入力）

ナビバーの **「記帳」** をクリックします。

### 基本的な入力手順

1. **取引日** を選択します（デフォルトは今日の日付）
2. **備考** に取引の内容を入力します（例：スーパー購入）
3. 仕訳行を入力します
   - **借/貸**：「借方」または「貸方」を選択
   - **勘定科目**：科目名を入力（途中まで打つとサジェストが出ます）
   - **金額**：半角数字で入力
4. **「記帳する」** ボタンをクリック

### 行の追加

**「＋ 行追加」** ボタンで行を増やせます。3行以上の複合仕訳にも対応しています。

### 金額の自動補完

1行だけ金額を空欄にしておくと、他の行から自動計算して補完されます（黄色でハイライト表示）。

**例**：食費 3,000円を現金で支払った場合
| 借/貸 | 勘定科目 | 金額 |
|-------|---------|------|
| 借方  | 食費    | 3,000 |
| 貸方  | 現金    | （空欄→自動補完） |

### 貸借バランスの確認

テーブル下部に借方合計・貸方合計がリアルタイム表示されます。
- 緑色：貸借一致（記帳可能）
- 赤色：貸借不一致（記帳不可）

### 記帳後

記帳が成功すると、同じ画面の右側（PC）または下部（スマホ）に確認テーブルが表示されます。  
「仕訳帳を確認」リンクから仕訳帳一覧に移動できます。

## 仕訳帳

ナビバーの **「仕訳帳」** をクリックします。

### 表示月の切替

画面上部のプルダウンで表示する月を切り替えます。  
記帳データがある月のみ選択肢に表示されます。

### 取引の確認

各行には取引日・備考・借方/貸方の科目名・金額が表示されます。  
**「詳細」** ボタンで各行の明細を展開表示できます。

### 取引の削除

**「削除」** ボタンをクリックすると確認ダイアログが出て、その取引の全行（複合仕訳を含む）をまとめて削除します。

> **注意**：削除したデータは元に戻せません。  
> 誤記帳の場合は、削除してから再入力してください。

### TSV エクスポート

画面右上の **「TSV エクスポート」** ボタンには3つのモードがあります。

| モード | 説明 | ファイル名 |
|--------|------|----------|
| 指定月のみ | プルダウンで選択中の月の仕訳 | `journal_YYYY-MM.tsv` |
| 全仕訳 | DB に登録されているすべての仕訳 | `journal_all.tsv` |
| 任意の期間 | 開始日〜終了日を指定してエクスポート | `journal_YYYY-MM-DD_YYYY-MM-DD.tsv` |

出力列：

| 列 | 内容 |
|----|------|
| 取引日 | YYYY-MM-DD 形式 |
| 備考 | 取引の備考 |
| 取引番号 | ファイル内での連番（複合仕訳は同じ番号） |
| 借貸 | 借方 / 貸方 |
| 勘定科目 | 科目名 |
| 金額 | 整数（円） |

Google スプレッドシートへのインポート手順：「ファイル」→「インポート」→「アップロード」→ 区切り文字「タブ」を選択。

## 財務諸表

ナビバーの **「財務諸表」** をクリックします。

### 表示月の切替

画面上部のプルダウンで月を選択します。

### 損益計算書（PL）

選択した月の収益・費用・当期純利益が表示されます。

| 項目 | 内容 |
|------|------|
| 収益合計 | 選択月の収益科目合計 |
| 費用合計 | 選択月の費用科目合計 |
| 当期純利益 | 収益合計 − 費用合計（マイナスの場合は赤字） |

### 貸借対照表（BS）

選択した月末時点の**累計残高**が表示されます。  
（その月だけでなく、記録開始からその月末までの全仕訳を集計します）

| 項目 | 内容 |
|------|------|
| 資産合計 | 資産科目の累計残高 |
| 負債合計 | 負債科目の累計残高 |
| 純資産合計 | 純資産科目の累計残高 ＋ 累計純損益 |
| 累計純損益 | 記録開始から選択月末までの収益−費用の累計。累計純損益の振替が登録されている場合は振替済み金額を除いた残額が表示される |

> **貸借差額の警告**  
> 資産合計 ≠ 負債合計 ＋ 純資産合計 の場合、表の下部に警告が表示されます。  
> 仕訳に誤りがないか確認してください。

### TSV エクスポート

画面右上の **「TSV エクスポート」** ボタンには3つのモードがあります。

| モード | 説明 | ファイル名 |
|--------|------|----------|
| 指定月のみ | 選択月の PL ＋ 月末時点の BS | `report_YYYY-MM.tsv` |
| 任意の期間 | 開始日〜終了日の PL ＋ 終了日時点の BS | `report_YYYY-MM-DD_YYYY-MM-DD.tsv` |
| 全月一覧（横展開） | 全月を1行1月の横展開形式で出力。BS は月末累計残高、PL は月次発生額 | `report_monthly_YYYY-MM_YYYY-MM.tsv` |

**全月一覧（横展開）の列構成：**

| 列 | 内容 |
|----|------|
| 決算月 | YYYYMM 形式 |
| 決算日 | YYYY/MM/DD 形式（月末日） |
| 当期純利益 | その月の収益−費用 |
| 資産合計 | 月末時点の資産累計残高 |
| 負債合計 | 月末時点の負債累計残高 |
| 純資産合計 | 月末時点の純資産累計残高（累計純損益含む） |
| 収益合計 | その月の収益合計 |
| 費用合計 | その月の費用合計 |
| 累計純損益 | 記録開始から当月末までの純損益累計。振替が登録されている場合は振替済み金額を除いた残額 |
| 各勘定科目 | 資産・負債・純資産は月末累計残高（純資産は振替分を含む）、収益・費用はその月の発生額 |

Google スプレッドシートへのインポート手順は仕訳帳と同様です。

## ダッシュボード

ナビバーの **「ダッシュボード」** をクリックします。

月次の財務データをグラフで把握するページです。上部に固定ウィジェット2つ、その下にユーザーが追加した予算実績ウィジェットが並びます。

---

### 固定ウィジェット①：比例縮尺損益計算書（月次）

借方（左）に費用科目と当期純利益、貸方（右）に収益科目と当期純損失を積み上げた棒グラフです。収益の規模と費用の構造を月ごとに横並べで比較できます。

- 期間セレクタ：3ヶ月 / 半年 / 1年（デフォルト）/ 全期間
- データ量に応じて横スクロール対応

---

### 固定ウィジェット②：純資産合計の推移

純資産合計（純資産科目の累計残高＋累計純損益）の月次推移を塗り付き折れ線グラフで表示します。振替が登録されている場合も合計値は正しく計算されます。

- 期間セレクタ：3ヶ月 / 半年 / 1年 / 全期間（デフォルト）

---

### 予算実績ウィジェット

費用科目と月次予算額を設定し、実績との差異を比較するグラフです。いくつでも追加できます。

#### グラフの読み方

| 位置 | 内容 |
|------|------|
| 借方（左） | 実績：費用科目の月次発生額を積み上げ |
| 貸方（右） | 予算：費用科目の月次予算額を積み上げ |

- Y 軸：各月の `max(実績合計, 予算合計)` を 100% に正規化
- **左が右より長い月** → 予算超過。予算バーの下端に**不利差異**（赤）が表示されます
- **右が左より長い月** → 予算内。実績バーの下端に**有利差異**（薄緑）が表示されます
- 実績の各科目バーをホバーすると、その科目の差異（有利 / 不利の別と金額）がツールチップに表示されます

#### 当月の予算按分

グラフを表示した日の日付を基準に、**当月分の予算だけ日数按分**して表示されます。

> 例：2月14日に開いた場合（2月が28日まである年）→ 当月予算 ＝ 設定額 × 14 ÷ 28

過去月は設定額のまま（按分なし）で表示されます。

#### ウィジェットの追加

1. ページ下部の **「＋ 予算実績グラフを追加」** をクリック
2. **タイトル** を入力（例：食費 予算実績）
3. ドロップダウンから **費用科目** を選んで「追加」を繰り返す
4. 各科目の **月次デフォルト予算額** を入力
5. **「保存」** をクリック

- 期間セレクタ：3ヶ月（デフォルト）/ 半年 / 1年 / 全期間

#### 月次予算の個別設定（任意）

モーダル内の「月次予算の個別設定」を展開すると、特定の月だけ予算額を上書きできます。指定しない月はデフォルト値が使われます。

| 入力項目 | 説明 |
|---------|------|
| 科目 | 上書き対象の費用科目 |
| 月（YYYY-MM） | 上書きする年月 |
| 金額（円） | その月の予算額 |

#### ウィジェットの編集・削除

- カード右上の **「編集」** ボタンで設定を変更できます（タイトル・科目・予算額・月次上書きの変更）
- **「削除」** ボタンでウィジェットをページから削除します

#### 権限について

予算実績ウィジェットの設定はプロジェクト単位で保存され、同じプロジェクトにアクセスするメンバー全員で共有されます。追加・編集・削除には **write 権限** が必要です。read 権限のユーザーはグラフの閲覧のみ行えます。

## 累計純損益の振替

財務諸表ページの下部にある **「累計純損益の振替」** セクションから操作します。

### 概要

pjkeep では会計期間はユーザーが自由に決められるため、決算のタイミングも自由です。  
この機能を使うと、任意の日付において「その前日までの累計純損益」を指定した純資産科目（例：繰越金）へ振り替えられます。

振り替えを行うと：
- 貸借対照表の「累計純損益」が振替済み金額の分だけ減る（または増える）
- 振替先の純資産科目の残高にその金額が加算される
- 純資産合計・資産合計・負債合計は変わらない（純資産内の組み替えのため）

この機能を使わないユーザーは何も操作しなくてよく、これまでと変わらない表示になります。

### 振替の登録

1. **「新規登録」** ボタンをクリックしてモーダルを開きます
2. **振替日** を入力します（例：2026-04-01）
   - 振替の根拠となる累計純損益は「振替日の前日まで」の集計です
   - 日付を入力すると **振替金額がリアルタイムで表示** されます
3. **振替先科目** を選択します（純資産区分の科目のみ選択可能）
4. 必要であれば **備考** を入力します
5. **「登録」** ボタンをクリックします

> **振替金額の計算**  
> 振替金額 ＝ 振替日前日までの全仕訳の収益−費用合計 − 過去の振替合計  
> つまり「前回振替後から今回振替日前日までの累計純損益」になります。

### 振替の削除

振替一覧の **「削除」** ボタンをクリックします。  
削除すると振替前の状態に戻り、BS 表示はすぐに更新されます。

### 仕様の詳細

| 項目 | 内容 |
|------|------|
| 同一日付の振替 | 1日1件のみ登録可能 |
| 振替先科目 | 純資産区分（`element = 'equity'`）の科目のみ |
| 振替金額がゼロ・マイナス | 登録可能。マイナスは赤字表示 |
| 金額の変更 | 削除して再登録で対応（金額は常に動的計算のため） |
| 遡及仕訳 | 振替日以前の仕訳を後から追加・修正した場合、振替金額は自動的に再計算される |
| P/L への影響 | なし。損益計算書の各科目・当期純利益の計算は振替の影響を受けない |

## 総勘定元帳

ナビバーの **「元帳」** をクリックします。

### 概要

特定の勘定科目に絞って、仕訳を日付順・残高推移付きで一覧表示するページです。  
銀行・クレカ明細との照合、特定科目の動きの追跡、AI への帳簿データ提供などに活用できます。

### 科目の選択

- **PC**：画面左のサイドバーに資産・負債・純資産・収益・費用の区分ごとに科目一覧が表示されます。科目名をクリックすると元帳が表示されます。
- **スマホ**：画面上部のプルダウンから科目を選択します。

### 元帳テーブルの見方

| 列 | 内容 |
|----|------|
| 日付 | 仕訳の取引日 |
| 備考 | 取引の備考 |
| 借方 | その行の借方金額（貸方行は空欄） |
| 貸方 | その行の貸方金額（借方行は空欄） |
| 残高 | その行時点の累計残高 |
| 相手科目 | 同一取引の相手勘定（複数なら「諸口」。PC のみ表示） |

**残高の正方向：**

| 科目区分 | 残高の正方向 |
|----------|------------|
| 資産・費用 | 借方超過 → 正 |
| 負債・純資産・収益 | 貸方超過 → 正 |

### 前期残高行

期間の開始日を指定した場合、その日より前の累計残高が「（前期残高）」として1行表示されます。  
前期残高がゼロの場合は表示されません。

### 期間フィルタ

開始日・終了日を入力して **「表示」** ボタンをクリックすると、指定期間のみ絞り込んで表示します。  
どちらも省略可能です（省略した側は全期間が対象になります）。

### 仕訳の編集・削除

元帳の行をクリックすると、その行が属する取引全体を編集するモーダルが開きます。

| 操作 | 説明 |
|------|------|
| 取引日・備考の変更 | モーダル上部の入力欄で変更し「保存」 |
| 仕訳行の変更 | 借/貸・科目・金額を変更し「保存」 |
| 行の追加・削除 | 「＋ 行を追加」ボタンで追加。各行の「×」で削除（最低2行） |
| 取引の削除 | モーダル左下の「削除」ボタン（この操作は取り消せません） |

保存・削除後は元帳ビューが自動的に再読み込みされます。

### TSV エクスポート

**「TSV エクスポート」** ボタンをクリックすると、現在表示中の科目・期間の元帳データをダウンロードできます。

| 列 | 内容 |
|----|------|
| 日付 | YYYY-MM-DD |
| 備考 | 取引の備考 |
| 取引番号 | 取引の識別番号（UUID） |
| 借方 | 借方金額（空欄または整数） |
| 貸方 | 貸方金額（空欄または整数） |
| 残高 | その行時点の残高 |
| 相手科目 | 相手の科目名（複数なら「諸口」） |

期間指定あり・前期残高が非ゼロの場合は、先頭に前期残高行（備考：「（前期残高）」）が出力されます。

**ファイル名：**

| 条件 | ファイル名 |
|------|----------|
| 期間指定なし | `ledger_{コード}_all.tsv` |
| 期間指定あり | `ledger_{コード}_{開始日}_{終了日}.tsv` |

**AI での活用例：**

```
以下の元帳データと銀行明細を照合し、
不整合の可能性がある行を指摘してください。

【普通預金 元帳（2026-01-01〜2026-03-31）】
（TSV をここに貼り付け）

【銀行明細（同期間）】
（明細 CSV をここに貼り付け）
```

## 勘定科目マスタ

ナビバーの **「勘定科目」** をクリックします。

### 科目の一覧

資産・負債・純資産・収益・費用の区分ごとに科目が表示されます。  
各科目には **科目名**、**コード**（英数字の識別子）、**表示順** があります。

### 科目の追加（1件）

**「＋ 科目を追加」** ボタンからモーダルウィンドウが開きます。

| 入力項目 | 説明 |
|---------|------|
| 科目名 | 表示に使う名前（例：食費）|
| コード | 英数字の識別コード（例：food）。自動生成されますが変更可能。|
| 区分 | 資産 / 負債 / 純資産 / 収益 / 費用 から選択 |
| 表示順 | 小さい数字が上に表示される（同一区分内での並び順）|

### 勘定科目一覧の JSON ダウンロード

**「ダウンロード (JSON)」** ボタンをクリックすると、現在登録されているすべての勘定科目を JSON ファイル（`accounts.json`）としてダウンロードできます。

出力形式はバッチ追加と同じ配列形式です。AI に仕訳を作成させる際の資料として使えます。

---

### 科目の一括追加（バッチ）

**「バッチ追加」** ボタンから、JSON 配列形式で複数科目をまとめて登録できます。

```json
[
  { "name": "交通費",  "code": "travel",      "element": "expenses",  "sort_order": 30 },
  { "name": "雑収入",  "code": "misc_income", "element": "revenues",  "sort_order": 30 },
  { "name": "長期借入金", "code": "loan",     "element": "liabilities", "sort_order": 30 }
]
```

**フィールド定義：**

| フィールド | 必須 | 型 | 説明 |
|-----------|------|----|------|
| `name` | 必須 | 文字列 | 科目名。DB 内で一意である必要があります |
| `code` | 必須 | 文字列 | 識別コード（英数字・アンダーバー推奨）。DB 内で一意 |
| `element` | 必須 | 文字列 | 区分（下表参照） |
| `sort_order` | 任意 | 整数 | 表示順（省略時は 0） |

**`element` の値：**

| 値 | 意味 |
|----|------|
| `assets` | 資産 |
| `liabilities` | 負債 |
| `equity` | 純資産 |
| `revenues` | 収益 |
| `expenses` | 費用 |

既に同名・同コードの科目が存在する行はスキップされ、他の行は登録されます。

### 科目の編集

各科目行の **「編集」** ボタンから変更できます。

### 科目の削除

**「削除」** ボタンをクリックします。  
その科目に仕訳が1件でも存在する場合は削除できません。

## バッチインポート（仕訳の一括取込）

ナビバーの **「バッチインポート」** をクリックします。

クレカ・銀行明細の自動仕訳、前期末の期首残高入力、給与仕訳など、まとめて取り込みたい場面で使います。  
**AI に元データ（明細 CSV・PDF など）と下記の JSON 仕様を渡すことで、自動的に仕訳 JSON を生成させることができます。**

### 操作の流れ

1. AI などで仕訳 JSON を作成する（後述）
2. バッチインポート画面の JSON テキストエリアに貼り付けるか、ファイルを選択する
3. **「プレビュー」** ボタンをクリック → バリデーション結果と仕訳内容が表示される
4. 取り込みたい仕訳にチェックを入れる（エラーのある仕訳は自動的に除外。二重記帳の可能性が高い仕訳はデフォルトでチェックが外れているので内容を確認する）
5. **「インポート実行」** ボタンをクリック → チェックした仕訳だけ DB に登録される

### JSON フォーマット仕様

#### 全体構造

```json
{
  "transactions": [
    {
      "date": "2026-03-15",
      "note": "Amazon 事務用品",
      "lines": [
        { "account_code": "operating", "debit_credit": "debit",  "amount": 3300 },
        { "account_code": "payable",   "debit_credit": "credit", "amount": 3300 }
      ]
    }
  ]
}
```

#### `transactions[]`（仕訳1件）のフィールド

| フィールド | 必須 | 型 | 説明 |
|-----------|------|----|------|
| `date` | 必須 | 文字列 | 仕訳日。`YYYY-MM-DD` 形式 |
| `note` | 推奨 | 文字列 | 摘要。仕訳帳の「備考」に入る |
| `lines` | 必須 | 配列 | 仕訳行（2行以上） |
| `_comment` | 任意 | 文字列 | AI の判断根拠など。DB には保存されず、プレビュー画面に表示されるだけ |
| `_confidence` | 任意 | 数値 | AI の確信度 0.0〜1.0。0.8 未満の場合はプレビューで黄色ハイライト表示。省略時は 1.0 扱い |

#### `lines[]`（仕訳行）のフィールド

| フィールド | 必須 | 型 | 説明 |
|-----------|------|----|------|
| `account_code` | 必須 | 文字列 | 勘定科目のコード（「勘定科目マスタ」画面の「コード」列の値） |
| `debit_credit` | 必須 | 文字列 | `"debit"`（借方）または `"credit"`（貸方） |
| `amount` | 必須 | 整数 | 金額（円）。1 以上の正の整数 |

> **`account_code` について**  
> 科目名（例：`事業費`）や科目 ID ではなく、コード（例：`operating`）で指定します。  
> コードは「勘定科目」画面の一覧で確認できます。

#### バリデーションルール（アプリが検証する）

1. `date` が `YYYY-MM-DD` 形式である
2. `lines` が 2 行以上ある
3. 各 `account_code` が勘定科目マスタに存在する
4. 各 `debit_credit` が `"debit"` または `"credit"` である
5. 各 `amount` が 1 以上の整数である
6. 1仕訳内の借方合計 = 貸方合計（複式簿記の大原則）

いずれかのルールに違反した仕訳はエラー表示され、チェックボックスが無効化されます（取り込みできません）。

### AI に渡すセット

| 渡すもの | 内容 |
|---------|------|
| このマニュアルの「バッチインポート」セクション | JSON フォーマット仕様とバリデーションルール |
| 勘定科目一覧 | 「勘定科目」画面の **「ダウンロード (JSON)」** ボタンで取得した `accounts.json`（コードと名前の対応が含まれます）|
| 元データ | クレカ明細 CSV・銀行明細・給与台帳・前期末 BS など |

### プレビュー画面の見方

| 表示 | 意味 |
|------|------|
| 緑枠（チェック済み） | バリデーション通過。インポート対象 |
| 赤枠「エラー」バッジ | バリデーション失敗。チェック不可 |
| 黄枠「確信度 XX%」バッジ | `_confidence` が 0.8 未満。AI が自信を持てなかった仕訳。内容を確認してください |
| 黄枠「⚠ 重複の可能性 XX%」バッジ | 既存データに金額・勘定科目構成が一致する仕訳が ±7 日以内に存在する。**デフォルトでチェックが外れています**。類似仕訳と照らし合わせて判断してください |
| 薄い黄枠「重複の可能性 XX%」バッジ | 上記と同様だが確率が 50〜79%。チェックはデフォルト ON のままです |

### 使用例

#### クレカ明細を仕訳する場合

AI に渡すプロンプト例：

```
以下の情報をもとに、pjkeep バッチインポート用の JSON を作成してください。

【仕様】
（このマニュアルの「バッチインポート」セクションをここに貼り付ける）

【勘定科目一覧】
（「勘定科目」画面の内容をここに貼り付ける）

【クレカ明細（2026年3月分）】
2026-03-10  Amazon  3,300円
2026-03-12  交通系IC  1,200円
...
```

#### 複数行仕訳の例（給与支払い）

```json
{
  "transactions": [
    {
      "date": "2026-03-25",
      "note": "3月分給与",
      "_comment": "給与台帳より。所得税・社会保険料を控除。",
      "lines": [
        { "account_code": "personnel",  "debit_credit": "debit",  "amount": 300000 },
        { "account_code": "deposit",    "debit_credit": "credit", "amount": 30000  },
        { "account_code": "bank",       "debit_credit": "credit", "amount": 270000 }
      ]
    }
  ]
}
```

## 期首残高の入力方法

新しく帳簿をつけ始めるとき、または年度をまたぐときに既存の残高を登録する方法です。

「前期繰越」として仕訳を入力することで記録します。  
バッチインポートを使うと、複数科目の残高をまとめて入力できます。

### 例：普通預金 500,000円 の残高から開始する場合

| 借/貸 | 勘定科目 | 金額 |
|-------|---------|------|
| 借方  | 普通預金  | 500,000 |
| 貸方  | 基本金   | 500,000 |

- 備考欄に「前期繰越」と入力しておくと分かりやすくなります
- 取引日は期首（例：2026-04-01）にします

### 複数の資産・負債がある場合

まとめて1つの仕訳として入力できます。

| 借/貸 | 勘定科目 | 金額 |
|-------|---------|------|
| 借方  | 普通預金  | 500,000 |
| 借方  | 現金     | 30,000 |
| 貸方  | 未払金   | 20,000 |
| 貸方  | 基本金   | 510,000 |

### バッチインポートで期首残高を入力する例

```json
{
  "transactions": [
    {
      "date": "2026-04-01",
      "note": "前期繰越",
      "lines": [
        { "account_code": "bank",     "debit_credit": "debit",  "amount": 500000 },
        { "account_code": "cash",     "debit_credit": "debit",  "amount": 30000  },
        { "account_code": "payable",  "debit_credit": "credit", "amount": 20000  },
        { "account_code": "capital",  "debit_credit": "credit", "amount": 510000 }
      ]
    }
  ]
}
```

## データのエクスポート

仕訳帳・財務諸表の各ページの **「TSV エクスポート」** ボタンからダウンロードできます。  
ファイルはタブ区切り・UTF-8（BOM なし）形式です。

### 仕訳帳エクスポートのモード

| モード | 内容 | ファイル名 |
|--------|------|----------|
| 指定月のみ | プルダウン選択中の月の仕訳 | `journal_YYYY-MM.tsv` |
| 全仕訳 | DB 内のすべての仕訳 | `journal_all.tsv` |
| 任意の期間 | 開始日〜終了日を指定 | `journal_YYYY-MM-DD_YYYY-MM-DD.tsv` |

### 財務諸表エクスポートのモード

| モード | PL の範囲 | BS の基準日 | ファイル名 |
|--------|---------|-----------|----------|
| 指定月のみ | 選択月 | 選択月末 | `report_YYYY-MM.tsv` |
| 任意の期間 | 開始日〜終了日 | 終了日時点 | `report_YYYY-MM-DD_YYYY-MM-DD.tsv` |
| 全月一覧（横展開） | 全月・月次 | 全月末（累計） | `report_monthly_YYYY-MM_YYYY-MM.tsv` |

### Google スプレッドシートへの取り込み手順

1. Google スプレッドシートを新規作成する
2. 「ファイル」→「インポート」をクリック
3. 「アップロード」タブからダウンロードした `.tsv` ファイルを選択
4. 区切り文字を **「タブ」** に設定してインポート

## データのバックアップ・移行

### バックアップ対象ファイル

| ファイル | 内容 | 場所 |
|---------|------|------|
| `*.db`（各プロジェクト） | 仕訳・勘定科目データ | `data/` 以下 |
| `users.db` | ユーザー情報・プロジェクト一覧 | `data/users.db` |

`data/` ディレクトリをまるごとバックアップすることを推奨します。

### ローカル版のバックアップ

```bash
cp -r /path/to/pjkeep/data /path/to/backup/data_20260401
```

プロジェクト単体をバックアップする場合：

```bash
cp /path/to/pjkeep/data/pjkeep.db /path/to/backup/pjkeep_20260401.db
```

### サーバ版のバックアップ（DB ダウンロード）

設定ページ（/init）の上部「現在のDB」欄にある **「↓ DBダウンロード」** ボタンをクリックすると、  
現在使用中の SQLite ファイルをダウンロードできます。  
ファイル名は `{DBファイル名}_{YYYYMMDD}.db` 形式になります。

ダウンロードした `.db` ファイルはローカル版 pjkeep でそのまま開くことができます。

### サーバ側のバックアップ（サーバ管理者向け）

```bash
cp -r /path/to/pjkeep/data /path/to/backup/data_20260401
```

`users.db` もバックアップ対象であることに注意してください。

### 別の PC への移行（ローカル版）

1. 元の PC の設定ページ（/init）の「↓ DBダウンロード」ボタンで `.db` ファイルをダウンロードします
2. 新しい PC で pjkeep を起動してログインします
3. 設定ページ（/init）の「**既存の .db ファイルをアップロードして開く**」からダウンロードしたファイルを選択してアップロードします

## トラブルシューティング

### 【サーバ版】起動後もセットアップ画面が表示され続ける

管理者アカウント作成後にセットアップ画面から抜け出せない場合は、ブラウザのキャッシュをクリアするか、シークレットモードで試してください。

### 【サーバ版】ブラウザで繋がらない

1. サーバの LAN IP を確認する：
   ```bash
   ip addr show | grep 'inet ' | grep -v '127.0.0.1'
   ```
2. Docker コンテナが起動しているか確認：
   ```bash
   docker compose ps
   docker compose logs nginx
   ```
3. ブラウザのアドレスバーに `http://IPアドレス:5000` の形式で入力しているか確認  
   （`https://` や `:5000` の省略などでアクセスできないことがあります）

### 【サーバ版】`docker compose` が使えない（`unknown command` エラー）

```bash
sudo apt-get install -y docker-compose-plugin
```

インストール後に `docker compose version` を実行してバージョンが表示されれば OK です。

### 【サーバ版】Docker が起動しない

```bash
docker compose logs app
docker compose logs nginx
cat .env  # SECRET_KEY が空でないか確認
```

### 【サーバ版】data/ のファイルが root 所有になる

`.env` に `APP_UID` と `APP_GID` が設定されているか確認してください。

```bash
# .env に追記してコンテナを再起動
echo "APP_UID=$(id -u)" >> .env
echo "APP_GID=$(id -g)" >> .env
docker compose up -d
```

### アプリが起動しない（ローカル版）

```
ModuleNotFoundError: No module named 'flask'
```
→ 依存パッケージがインストールされていません。
```bash
pip3 install -r requirements.txt
```

### ログインできない

- ユーザー名・パスワードを正確に入力しているか確認してください
- 初回アクセス時に管理者アカウントを作成していない場合は `/setup` にアクセスしてください
- パスワードを忘れた場合は、サーバ管理者が `data/users.db` を直接操作するか、`users.db` を削除して再セットアップしてください（既存の帳簿データは消えません）

### 「DBが設定されていません」エラー

- 設定ページ（/init）からDBを作成または選択してください
- 別のブラウザや別の端末からアクセスした場合は、それぞれのブラウザで DB を選択し直す必要があります（どの DB を開くかはブラウザの Cookie で管理されているため）

### 「アクセス権がありません」エラー

そのプロジェクトへの参加権限がありません。  
設定ページ（/init）から自分のプロジェクトを選択してください。  
admin に依頼して権限を付与してもらうか、admin ロールであればすべてのプロジェクトにアクセスできます。

### 「編集権限がありません」エラー

そのプロジェクトへの参加権限が `read`（閲覧のみ）に設定されています。  
仕訳の追加・編集・削除、勘定科目の変更、バッチインポートなどの書き込み操作を行うには `write` 権限が必要です。  
admin に `write` 権限への変更を依頼してください。

### 勘定科目が削除できない

仕訳に使用されている科目は削除できません。  
その科目を使っている仕訳をすべて削除してから、再度お試しください。

### バッチインポートで「account_code が存在しません」エラーが出る

AI が使用したコードが勘定科目マスタに登録されていません。  
「勘定科目」画面で該当のコードを確認し、必要であれば科目を追加してから再度インポートしてください。
